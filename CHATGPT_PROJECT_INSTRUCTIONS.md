# ChatGPT Project Instructions

この ChatGPT Project では `ai-shared-memory` を共有知識ベースとして扱う。

Repository:
https://github.com/bellbullet/ai-shared-memory

基本方針:

- まず `CHATGPT_PROJECT_SOURCE.md` の要約を前提にする。
- 詳細が必要な場合は `PROJECT_START.md` → `WORKSPACE.md` → `STATUS.md` → `AGENTS.md` → `CURRENT.md` → `AI_INDEX.md` の順で確認する。
- 非自明な作業の前に `AI_INDEX.md`、`AI_DRAWERS.md`、関連する `PROJECTS/` / `NOTES/` を確認し、`Found / Reuse / Not used / Reason` を短く示す。積極利用する候補は原則3つまでにする。
- 登録済み候補を実際に試した場合は、結果、判断、再利用条件を `TRIALS.md` への反映候補として提案する。
- GitHub URLだけで repo 全文を読めていると仮定しない。必要なファイルが見えていない場合は、ユーザーに該当ファイルの内容提供を依頼する。
- 新しい GitHub link、OSS、paper、tool、AI記事は、公開可能性を確認し、`AI_DRAWERS.md`、`INBOX.md`、`NOTES/`、`PROJECTS/`、`LESSONS_LEARNED.md`、`DECISIONS.md` のいずれに反映すべきか提案する。
- 長期的価値がある内容は、会話だけに残さず `ai-shared-memory` への反映候補として提案する。
- API key、token、cookie、webhook、`.env` の実値、認証情報、個人情報、住所、電話番号、契約内容、見積金額、ローカル端末固有ログは保存・出力しない。
- 事実、推測、アイデアを分けて書く。
- 既存構成を壊さず、変更は最小差分にする。
