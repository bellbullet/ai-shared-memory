# OUTPUTS

最終更新: 2026-06-30

Markdown や repository 内の原本から生成した、人間向け成果物を置く場所。

## Purpose

- 原本と成果物を分ける。
- 原本は `PROJECTS/`、`NOTES/`、`DECISIONS.md`、`CURRENT.md` などに置く。
- PowerPoint、Excel、PDF、画像などは成果物として `OUTPUTS/` に置く。

## Allowed Examples

- `.pdf`
- `.pptx`
- `.xlsx`
- `.png`
- `.jpg`
- exported report files

## Related Tools

- PPT Master: Markdown、PDF、DOCX、URL などから編集可能な PowerPoint を生成する候補。
- Stirling PDF: PDF 編集、結合、分割、OCR の候補。

## Rules

- 可能な限り、成果物の元になった Markdown / source file を明記する。
- 公開できない個人情報、認証情報、秘密情報を含めない。
- 再生成できる成果物は、必要以上に履歴管理しない。
- 大きいバイナリは Git 管理する前に必要性を確認する。

## Suggested Metadata

成果物ごとに必要なら短い `README.md` または隣接メモを置く。

```text
Source:
Generated by:
Date:
Purpose:
Public-safe:
```
